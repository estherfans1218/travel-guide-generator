# 旅游攻略生成器 — AI Agent 通用指令

本文件是一份**跨平台通用指令**，任何具备联网搜索和文件写入能力的 AI agent（WorkBuddy、Claude Code、Codex、Trae、Cursor 等）读取后均可执行。不绑定特定工具名或触发机制。

## 如何使用（各 agent）

| Agent | 用法 |
|-------|------|
| **WorkBuddy** | 本目录放到 `~/.workbuddy/skills/`，SKILL.md 自动触发 |
| **Claude Code** | 将本目录放入项目，`CLAUDE.md` 会被自动读取 |
| **Codex** | `AGENTS.md` 会被自动读取 |
| **Cursor** | `.cursorrules` 会被自动读取 |
| **Trae / 其他** | 让 agent 读取本文件 `AGENT_GUIDE.md` 即可 |

## 任务

根据用户指定的目的地及出行参数，用联网搜索能力获取（优先参考小红书热门推荐）实时信息，生成旅游攻略正文（Markdown，含图片占位符）。默认只输出攻略正文；当用户明确用于闲鱼等平台售卖时，额外生成上架文案和差异化补充清单。

## 输入参数

从用户输入中提取，缺失时按默认值或主动询问（最多问 2-3 个关键项，其余用默认）：
- 目的地（必需，必须确认）
- 天数（可选，默认 5 天）
- 出发地（可选，影响大交通推荐）
- 到达时间（可选，日期+时段如"下午2点"；影响首日行程）
- 离开时间（可选，日期+时段；影响末日行程）
- 到达/离开交通方式（可选：飞机/高铁 + 具体机场或车站名）
- 人均预算（可选，默认不限定）
- 同行人群（可选：单人/情侣/亲子/家庭/朋友，默认通用）
- 主题偏好（可选：美食/摄影/文化/休闲/户外，默认综合）
- 用途（可选：自用/售卖，默认自用；选"售卖"时额外生成上架文案和差异化补充清单）

## 工作流程

### 1. 联网搜索

用你的联网搜索能力，针对目的地搜索以下维度，**优先参考小红书热门推荐**（搜索关键词中加入"小红书"）：
- 必去景点与门票/开放时间
- 城际与市内交通（机场/高铁/地铁/打车/包车）
- 住宿区域推荐与价位
- 本地美食与推荐店铺
- 最佳旅行季节与天气
- 注意事项与常见坑（骗局、宰客、高反等）
- 签证/证件要求（出境目的地）
- 若提供了出发地，额外搜索"{出发地} 到 {目的地} 高铁/航班 价格"

详细的搜索维度与关键词见 `references/search-strategy.md`。

搜索要点：
- 多源交叉验证，优先采信近 1 年内的信息
- 对具体店铺名、票价、营业时间等易变信息，用网页抓取能力核实近期攻略或官方页面
- 当多个来源（尤其小红书）推荐同一店铺/机位时，可信度高，优先采用
- 不编造搜索不到的内容

### 2. 整理生成

按 `references/guide-structure.md` 的标准结构整理攻略正文。若用途为"售卖"，另按 `references/listing-copy.md` 生成上架文案。写作要求：
- 真实准确：基于搜索结果，不编造不存在的店铺/路线/票价
- 优先小红书：推荐（餐厅、景点、机位等）优先采用小红书热门推荐中口碑好的内容
- 图片占位符：每个核心景点/美食/住宿处预留图片占位符，格式 `![在此插入：XX实拍图](图片路径)`，供用户填入自己的实拍图
- 行程适配时间：根据到达/离开时间定制首末日行程（如下午到达则首日只排下午晚上）
- 大交通定制：若提供出发地和交通方式，交通板块优先写从该出发地出发的具体班次/价格建议
- 标注时效：票价、营业时间等易变信息标注"截至整理时，建议出行前再核实"
- 实用优先：每个板块都要有可操作的具体信息（具体到店名、地址、价格、怎么去）
- 详略得当：核心景点详写，次要景点简列

**重要**：攻略正文中不含"上架前建议补充"之类的话——那是给卖家自己的提醒，仅在售卖模式下单独放到"差异化补充清单"文件。

### 3. 输出文件

**默认输出**（任何用途都生成）：
- `{目的地}-旅游攻略.md` — 攻略正文，含图片占位符，纯攻略内容

**可选输出**（仅当用途为"售卖"时额外生成）：
- `{目的地}-上架文案.md` — 闲鱼上架用，含标题/封面文案/详情描述/定价建议/标签关键词
- `{目的地}-差异化补充清单.md` — 给卖家自己看，列出需补充的实拍图清单、踩坑经历、花费明细、小众机位等

保存到当前工作目录，告知用户生成的文件路径，并简要说明各文件用途。

## 写作规范

- 语言：简体中文
- 语气：像有经验的朋友在支招，实用、接地气，不堆砌辞藻
- 长度：攻略正文 6000-10000 字为宜
- 格式：标准 Markdown，用二级/三级标题分板块，善用表格（预算表、交通表）和列表
- 图片：用 `![描述](路径)` 占位符，描述写明"在此插入：XX实拍图"，不抓取网络图片
- 价格：统一标注货币单位，国内用 ¥，境外用当地货币 + ¥ 参考价
- 涉及边界/主权表述（如港澳台）：严格使用"中国香港""中国台湾""中国澳门"等规范表述

## 注意事项

- 图片只能用占位符，不抓取网络图片（版权风险，用于售卖会被举报）
- 小红书原文有反爬，以搜索摘要为准，无法逐篇抓全文
- 不编造信息：搜索不到的内容宁可省略或标注"建议自行了解"
- 出境目的地需提示签证/证件要求，但不替代官方信息，提示以官方为准
- 上架文案和差异化补充清单是可选功能，仅当用户明确用于售卖时生成
