# 攻略标准结构

本文件定义生成旅游攻略正文时的标准结构和各板块写作要求。生成时按此结构组织，可根据目的地特点适当增减板块。

## 标题

格式：`{目的地}{天数}天{主题}深度游攻略｜{一句话卖点}`
示例：`成都5天4晚深度游攻略｜人均2500吃遍地道老街`

注意：标题不含"售卖""上架"等字样，这是给买家看的纯攻略。

## 开篇说明（置顶）

简短说明，包含：
- 本攻略适用人群与季节
- 时效性提示：票价/营业时间截至整理日期，出行前请再核实

注意：不在此处写"上架前建议补充"等内容（那是给卖家自己的，放在单独的差异化补充清单文件）。

## 图片占位符规范

攻略中每个核心内容处预留图片占位符，供卖家填入自己的实拍图。格式：
```
![在此插入：XX实拍图](图片路径)
```
放置位置：
- 每个核心景点详写处放 1 个占位符（如"在此插入：火炬八街实拍图"）
- 每个推荐餐厅处可放 1 个占位符（如"在此插入：蒸汽海鲜实拍图"）
- 住宿推荐处可放占位符
- 行程规划每天可放 1 个占位符（当日代表场景）

占位符不抓取网络图片，只标注提示，由卖家自行填入实拍图。

## 一、目的地概览

- 一段话简介目的地特色与适合什么样的人
- 最佳旅行季节与天气
- 建议游玩天数

## 二、行程规划（核心）

按天拆解，**首末日根据到达/离开时间定制**：
- 若到达时间为下午/傍晚：首日只安排下午+晚上（抵达接驳+入住+附近晚餐/夜市）
- 若离开时间为上午：末日只安排上午（早餐+购物+前往车站/机场）
- 中间天数按全天安排

每天包含：
- 当日主题与路线
- 上午/下午/晚上具体安排
- 景点间交通方式与耗时
- 当日用餐推荐
- 当日住宿建议（如换住宿点）
- 当日代表场景图片占位符

示例格式：
```
### Day 1：抵达 + 市区初探（下午到达）
- 14:00 抵达XX机场/高铁站，前往酒店入住...
- 下午：游览XX景点（门票¥X，建议X小时）...
![在此插入：XX景点实拍图](图片路径)
- 晚上：XX美食街晚餐...
- 住宿：建议住XX区域
```

## 三、交通指南

- 大交通：如何抵达（机场/高铁/火车），各方式优劣与价格
  - 若用户提供了出发地，优先写从该出发地出发的高铁班次/航班/价格建议
  - 若用户指定了交通方式（飞机/高铁）和具体站，重点写该站到市区的接驳
- 市内交通：地铁/公交/打车/包车/租车，各自适用场景与价格
- 景点间交通：重点标注易踩坑的路线（如黑车、绕路）
- 用表格呈现主要交通方式与价格

## 四、住宿推荐

- 按区域推荐（不同区域的特点与适合人群）
- 每个区域给 2-3 个价位档次的建议（青旅/经济连锁/中端/高端）
- 标注预订建议（提前多久、平台、避坑）
- 住宿处可放图片占位符

## 五、美食推荐

- 必吃特色（带简短介绍）
- 推荐店铺（店名、地址/区域、人均、招牌菜、避坑提示）——优先小红书热门推荐
- 按价位或类型分类
- 标注哪些是游客必去、哪些是本地人常去
- 推荐餐厅处可放图片占位符

## 六、景点详解

- 核心景点详写：门票、开放时间、游玩时长、亮点、避坑、拍照机位（优先小红书机位推荐）
- 次要景点简列：一句话推荐 + 门票
- 用表格汇总所有景点信息
- 每个核心景点放图片占位符

## 七、避坑指南（卖点板块）

- 常见骗局与宰客套路（优先参考小红书"踩雷"内容）
- 交通避坑
- 住宿避坑
- 购物避坑
- 季节/天气相关注意（如雨季、高反、台风）
- 本地人不会做的事

## 八、预算明细

- 按人均列出各项预算（交通/住宿/餐饮/门票/其他）
  - 若提供了出发地，大交通预算按从该出发地的实际票价估算
- 给出经济/舒适/品质三档预算参考
- 用表格清晰呈现
- 标注哪些可省、哪些别省

## 九、实用贴士

- 必备物品清单
- 证件与支付
- 通讯与网络
- 紧急联系方式
- 当地习俗与禁忌
- App 推荐

## 写作要求

- 每个板块都要有具体可操作的信息，不要空话
- 价格、时间、地址等关键信息要具体
- 避坑指南是差异化卖点，要写得真实有用
- 表格用于汇总对比信息，列表用于步骤/清单
- 核心板块（行程、避坑、预算）详写，辅助板块简明
- 图片占位符分布合理，核心景点/美食/住宿处都要有，但不要过密
